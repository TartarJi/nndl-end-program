# nndl-end-program
西南交通大学 神经网络与深度学习B 课程大作业

选取经典论文'Attention Is All You Need'复现，实现了一个完整的 Seq2Seq（序列到序列）翻译模型，原创为：GitHub 用户 jadore801120，本人完成学习和实践

以下是项目的完整流程

---

### 第一步：准备运行环境

首先需要把项目从 GitHub 上克隆到本地。然后创建一个独立的 Python 环境，避免和其他项目产生包版本冲突。环境创建好并激活之后，安装 PyTorch 深度学习框架，以及几个关键的依赖库，包括处理自然语言数据的 torchtext、用于分词和词性标注的 spaCy，还有 tqdm 进度条和 dill 序列化工具，版本要求在request.txt。接着要用 spaCy 自带的命令行工具下载英语和德语的语言模型，这两个模型会在后续数据预处理阶段被用来对文本进行分词和词性标注。

---

### 第二步：数据预处理

运行项目提供的预处理脚本，指定源语言为德语、目标语言为英语，同时开启共享词汇表的选项，使编码器和解码器会使用同一套词汇表，有助于提升模型性能。预处理脚本会从网上下载 WMT'16 多模态翻译数据集，然后用 spaCy 对每个句子进行分词，接着构建词汇表，把单词映射成数字编号。处理完成后，所有数据会被打包保存成一个 pickle 格式的文件，供后续训练直接加载使用，这样就不用每次都重复预处理。

---

### 第三步：模型训练

运行训练脚本，指定刚才生成的 pickle 数据文件作为输入。为了训练出好的效果，可以开启几个论文中推荐的技巧：一是编码器和解码器共享嵌入层权重，二是输出投影层也共享权重，三是使用标签平滑来防止模型过于自信，这些都能提升翻译质量。还需要设置批次大小、预热步数和训练轮数。如果只是想快速验证程序能不能跑通，可以把训练轮数设为 2 到 5 轮。因为代码版本太过久远，pytorch版本并不适用于本人电脑的gpu，因此使用cpu来计算，10轮的训练时间约为3h。

---

### 第四步：用训练好的模型做翻译

训练完成后，运行翻译脚本，指定之前预处理生成的 pickle 数据文件和训练过程中保存的模型检查点文件，再指定一个输出文本文件的路径。脚本就会加载模型，对输入的数据进行翻译，最后把翻译结果写入指定的文本文件里，这样就完成了一个完整的从数据准备到模型训练再到翻译验证的流程。

---

### 遇到的主要问题

整个流程中有几个地方容易出问题。比如数据集下载失败，因为原始 WMT 数据链接可能已经失效，这时需要手动找可用的数据源或者换成其他数据集。其次是 torchtext 库的版本问题，不同版本之间 API 变化很大，可能需要调整导入语句或者安装特定版本。再就是显存不足的问题，需要通过减小批次大小或者简化模型来解决。另外 spaCy 语言模型如果下载的版本和代码里调用的名字不匹配，也会报错，需要确认下载了正确的模型名称。
